## Version 0.1.4-dev — Pre-Alpha
**Status:** Completed
**Branch:** 0.1.4-dev

This release marks the transition into the **pre-alpha** stage. Core architecture for node management and server lifecycle is now in place, bringing the project close to its initial full realization.

---

## Yunexal Panel

### UI / UX
- Fixed server creation flow for Images that do not require port allocations.
- Dynamic handling of “Default Allocation” (Auto-Assign / None) based on Image configuration.

### Runtimes (Nests)
- Implemented full CRUD support.
- Added customizable hex color field.
- Displayed color indicators in the Runtime list.
- Added dedicated edit page for Runtime management.
- Implemented HTMX-based delete actions.
- Added automatic database migration for `color` field.
- Implemented drag-and-drop ordering.
- Added Disk I/O (read/write) monitoring.
- Added dashboard auto-refresh (5s).
- Added cluster-wide resource overview.

### Images (Eggs)
- Implemented full Image creation workflow under Runtimes.
- Added advanced configuration support:
  - `startup_command`
  - `stop_command`
  - `docker_images`
  - `log_config`
  - `config_files`
  - `start_config`
- Implemented full Pterodactyl Egg import (JSON):
  - Docker images
  - Start/stop lifecycle configuration
  - Log configuration
  - Installation scripts
  - Environment variables (stored as JSON)
- Added expandable Image list in Runtime view.
- Added inline Image edit and delete actions.
- Unified Image creation flow (manual or import).

### Customization
- Added configurable panel font via `.env` (default: Google Sans Flex).
- Added Open Font License file (`OFL.txt`).

---

## Yunexal Node

- Implemented full Docker container lifecycle management (create, stop, delete).
- Added 10-second grace period before forceful container removal.
- Added pre-flight port availability checks.
- Implemented WebSocket console endpoint (`/console`) with direct TTY access.
- Compiled optimized release binaries.
- Implemented consolidated 3×2 dashboard grid layout.
- Added Network I/O monitoring (Rx/Tx).
- Added detailed per-disk usage monitoring.
- Implemented smart disk grouping for physical devices.
- Updated disk icons for SSD/NVMe and HDD.
- Added global “Connection Lost” banner.
- Added global footer with execution time.
- Changed default panel port to `65535`.
- Fixed layout issues and resolved compiler warnings.

---

## Fixes
- Resolved allocation-related UI inconsistencies.
- Updated dependencies for WebSocket support.
- Removed unused code and resolved compiler warnings.
